# 02-Techniques Summary

本章节深入探讨了 Web 逆向的核心技术与方法论。

---

## 💭 开始前的思考

在学习具体技术之前，先停下来思考：

1. **为什么需要 Hook？** 直接看源码、打断点不行吗？
2. **如果你是攻击者**，会在哪些地方设置"陷阱"阻止逆向分析？
3. **混淆代码的目的是什么？** 真的能完全阻止逆向吗？
4. **调试和逆向的本质区别是什么？** 为什么说"逆向是一门艺术"？

这些问题的答案，都藏在本章的技术细节中。

---

## 主要内容

- **Hook 技术**:
  - **API Hook**: 劫持 XHR, Fetch, JSON.parse, JSON.stringify 等关键函数。
  - **Prototype Hook**: 通过修改原型链拦截对象方法。
  - **Property Hook**: 使用 `Object.defineProperty` 拦截属性读写。
- **调试技巧**:
  - 条件断点、XHR 断点、DOM 断点。
  - Local Overrides (本地替换): 替换线上文件进行调试。
- **代码混淆与反混淆**:
  - 常见混淆手段 (变量名替换、控制流平坦化、字符串加密)。
  - AST (抽象语法树) 基础与还原脚本编写。
- **加密参数定位**:
  - 关键词搜索、堆栈跟踪 (Call Stack)、跟值分析。
