# 04-Engineering Summary

本章节关注逆向工程的自动化与体系化建设。

---

## 💭 开始前的思考

在学习工程化之前，思考这些实际问题：

1. **手工逆向 vs 工程化：** 为什么逆向出加密算法后，还要投入精力做工程化？
2. **扩展性挑战：** 如果你需要每天爬取 100 万条数据，手工脚本能撑住吗？
3. **维护成本：** 网站更新了反爬虫策略，你的脚本要从头改吗？
4. **真实场景：** 你逆向出了某网站的签名算法，现在需要在服务器上批量调用。是把 JavaScript 代码搬到 Python 里重写，还是有更优雅的方案？

工程化不是炫技，而是让逆向成果真正落地的桥梁。

---

## 主要内容

- **RPC 技术**:
  - Sekiro / JsRPC 等框架使用，实现从浏览器到本地代码的远程调用。
- **自动化脚本**:
  - Puppeteer / Playwright 自动化控制。
  - 补环境框架 (JSDOM / Node.js Mock Web API)。
- **大规模采集架构**:
  - 代理池管理。
  - 指纹池管理。
